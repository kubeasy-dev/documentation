The `kubeasy` CLI is the main tool to interact with Kubeasy locally: start challenges, submit solutions, reset environments, and more.

Built with Go and powered by [Cobra](https://cobra.dev/), the CLI is distributed as a single binary with no dependencies.

**Installation:**
Download the latest release for your platform from [GitHub Releases](https://github.com/kubeasy-dev/kubeasy-cli/releases) or see the [Getting Started guide](/docs/user/getting-started) for detailed installation instructions.

```bash
kubeasy version
```

### `kubeasy challenge reset <challenge>`

Reset the challenge environment by reapplying the initial manifests. This will also reset your progress and delete your submission on the platform.
Use this if you're stuck or want to start fresh.

**Usage:**
```bash
kubeasy challenge reset <challenge-name>
```

**Example:**
```
$ kubeasy challenge reset rbac-basics
âš ï¸  This will reset the challenge to its initial state.
   Your current progress will be lost.

   Continue? (y/N): y

ğŸ”„ Resetting challenge...
âœ“ Deleted existing resources
âœ“ Reapplied initial manifests
âœ“ Challenge reset complete

You can now try solving it again from scratch.
```

**Options:**
- `--force` - Skip confirmation prompt
- `--keep-submission` - Reset cluster resources but keep your submission history

**Example with options:**
```bash
kubeasy challenge reset rbac-basics --force --keep-submission
```

### `kubeasy challenge clean <challenge>`

Clean up all Kubernetes resources related to the challenge, including its namespace.
This does **not** affect your challenge progress on the platform.

**Usage:**
```bash
kubeasy challenge clean <challenge-name>
```

**Example:**
```
$ kubeasy challenge clean rbac-basics
ğŸ§¹ Cleaning up challenge resources...
âœ“ Deleted ArgoCD Application
âœ“ Deleted namespace 'challenge-rbac-basics'
âœ“ Cleanup complete

Your challenge progress on the platform remains intact.
```

**When to use:**
- You've finished the challenge and want to free up resources
- You want to remove a challenge without resetting your submission
- You're switching to a different challenge

**Clean up all challenges at once:**
```bash
# List all active challenges
kubeasy challenge list --active

# Clean them all
for challenge in $(kubeasy challenge list --active --format=names); do
  kubeasy challenge clean $challenge
done
```
